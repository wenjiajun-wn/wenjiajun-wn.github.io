什么是 FastAPI？

FastAPI 是一个现代、高性能的 Python Web 框架，专门用于构建 API。它由 Sebastián Ramírez 开发，基于以下两个强大的库：

  • Starlette：提供异步 Web 框架的核心能力
  • Pydantic：负责数据验证与序列化

FastAPI 的名字本身就说明了它的两大特点：Fast（快速）—— 既指运行速度快，也指开发速度快。

为什么选择 FastAPI？

特性 说明
高性能 性能可与 Node.js 和 Go 媲美
自动数据验证 基于 Python 类型提示自动校验请求数据
自动生成文档 内置 Swagger UI 和 ReDoc，无需手写文档
Python 类型提示 充分利用 Python 3.6+ 的类型系统
易于上手 代码简洁直观，

九、核心概念

总结

FastAPI 最大的魅力在于：用最少的代码，做最多的事。

只需定义路由函数 + Pydantic 模型，FastAPI 自动搞定：

  • 参数解析 & 类型转换
  • 数据合法性校验
  • JSON 序列化 / 反序列化
    (在 FastAPI 里你不需要手动做这些——你返回一个 dict，它自动序列化成 JSON；前端发来 JSON，它自动反序列化成 Pydantic 对象。简单说就是”打包发送、拆包接收”，FastAPI 帮你全自动搞定。)
  • Swagger（现在叫 OpenAPI） 文档自动生成(简单说就是自动给你的接口生成一个可视化页面能在浏览器里查看和测试所有 API。)

第二节：FastAPI 进阶主题

一、PUT 与 DELETE 操作

在第一节里，我们学会了用 GET 读取数据、用 POST 创建数据。现在有另外两种 HTTP 操作：

  • PUT：更新已有对象
  • DELETE：删除已有对象

和 POST 一样，它们都可以从查询字符串（query string）和请求体（request body）中接收参数。关键区别在于——这两种操作需要引用一个已存在的对象。

引用已有对象

FastAPI 没有内置 ORM，这意味着把 API 请求映射到数据库对象是应用本身的责任。惯例做法是在参数名里包含列名，例如 review_id 表示 reviews 表的主键 ID。

二、错误处理与 HTTP 状态码

一个好的的 API 不能只处理顺利的操作，还必须要处理各种错误，并始终返回恰当的 HTTP 状态码。

状态码

范围 含义 典型示例
2xx 成功 200 OK、201 Created、204 No Content
4xx 客户端错误 400 Bad Request、404 Not Found、422 Unprocessable Entity
5xx 服务器错误 500 Internal Server Error

三、使用 async 处理并发请求

现代 API 往往需要同时处理大量请求。FastAPI 支持 Python 的 async/await 语法，让你的应用能以更低的消耗应对更高的并发。

什么时候用 async？

  • 调用外部 API 或数据库时（I/O 密集型操作）
  • 操作本身有等待时间，而 CPU 其实处于空闲状态时

如果函数只做纯 CPU 计算（不涉及 I/O 等待），则不需要 async。

异步 DELETE 功能

加上 async def 和 await，FastAPI 就能在等待数据库响应时继续处理其他请求，显著提升吞吐量。
