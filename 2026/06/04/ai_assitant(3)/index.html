认证方式

项目中有两种认证方式：

  • 浏览器 → Cookie + Session（登录后自动）
  • exe 客户端 → X-API-Token Header（注册时自动获取并保存）
  • 注册界面

Cookie

Cookie 是浏览器里存的一小段数据，每次发请求时自动带在 HTTP Header 里。例如：

特点：

  • 存在用户浏览器端
  • 每次请求自动携带，不需要代码手动加
  • 可以设置过期时间

Session

Session 是服务端存的数据，用来记住”这个用户是谁、登录了没有”。例如：

特点：

  • 存在服务器端（Flask 默认加密存在 Cookie 里，也可以存 Redis/数据库）
  • 用户拿不到、改不了，安全
  • 登出时 session.clear() 删掉即可

Cookie + Session 的配合流程：

  1. 用户输入密码登录
  2. 服务器验证通过 → 创建 Session（存用户信息）→ 返回一个 Session ID
  3. 浏览器把 Session ID 存到 Cookie 里
  4. 之后每次请求，浏览器自动在 Cookie 里带上 Session ID
  5. 服务器根据 Session ID 查到 Session 数据 → 知道是谁在操作

Redis

Redis 是一个内存数据库，数据存在内存里而不是硬盘上，所以读写极快。

Session 的存储方式之一就是 Redis：

存储方式 特点
Flask 默认（加密 Cookie） 简单，但数据量受限
Redis 快、支持分布式、服务器重启数据不丢
SQLite/MySQL 持久化，但比 Redis 慢

简单记住：Cookie 是浏览器存的通行证，Session 是服务器存的用户档案，Redis 是存放这些档案的高速柜子。
