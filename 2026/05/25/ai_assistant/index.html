一个可以提取图片中重要信息并分成优先级的ai助手

项目概述

日常工作中截图里有很多待办信息，手动录入日历太麻烦，所以想用 AI 自动化。

项目的本质是：用户截图或上传图片 → OCR 提取文字 → 交给 AI 大模型分析总结 → 最终提取出待办事项。

技术选型

OCR（光学字符识别，Optical Character Recognition）是一种能把图片、扫描件或纸质文档上的文字”翻译”成电子设备可以编辑、搜索的文本的技术。

我们需要强大的 OCR 来处理图片，因此选择了 PaddleOCR。相比传统 OCR 工具，PaddleOCR 的优势在于：

  • 基于深度学习架构，识别精度很高
  • 支持的语言库庞大
  • 传统的 OCR 只能逐字读出文字，而 PaddleOCR-VL 系列模型可以看懂文档的复杂排版
  • 极其轻量化，部署成本低

下一步就是把提取出来的文字交给 AI 大模型分析，比如 DeepSeek、ChatGPT 等。需要注意的是，如果使用 ChatGPT 可以简化 OCR 步骤，因为它具有 Vision（视觉）模型，能直接从图片中识别文字。AI 分析后会返回 JSON 格式的结构化数据，例如：

返回 JSON 的好处是结构化——程序可以直接用 对象.属性 或 数组[索引] 的方式读取数据，前后端都能轻松处理，更加轻量化。

前端使用 Streamlit，它是一个开源的 Python 框架，可以快速将 Python 脚本转化为交互式 Web 应用。只需 pip install streamlit，然后使用内置函数就能创建页面和交互组件：

  • st.title：定义页面标题
  • st.button：触发交互操作
  • st.dataframe：展示核心数据表格

剪贴板与图片预处理

ImageGrab.grabclipboard() 是 Windows 专有 API，能直接从剪贴板拿到 PIL Image 对象——不需要保存文件到磁盘，也不需要用户选择路径。

MD5 去重：MD5 是一种哈希算法，能将任意长度的数据（比如一张图片、一段文字）计算成一个固定 128 位（32 个十六进制字符）的”指纹”。Windows 截图工具对同一张图不会重复放入剪贴板，但微信/QQ 截图可能会。对整张图的 tobytes() 做 MD5，比存 PIL Image 对象更省内存。

图片预处理（pil_to_base64）：

  • 长边超过 2048px 等比缩放，控制传输体积
  • RGBA/P 模式转 RGB，避免 JPEG 编码报错
  • quality=85，在体积和清晰度之间取平衡

processing 标志：防止 AI 分析期间（可能 5-10 秒）重复触发。外层 while True 捕获所有异常，内层 last_hash = None 重置确保出错后下一次能重新尝试。

Vision LLM 与 Prompt

关于 Vision LLM 的选择：刚开始试用的是 DeepSeek，后来发现它不是多模态模型，于是换成了阿里云的 qwen-vl-max。也可以使用 GPT-4V 等，但需要注意 GPT 的 Vision 模型需要网络代理，而且成本较高。

Prompt 的设计思路：

  • 逐区域扫描 → 列出每个任务 / @提及 / 会议 / 截止日期 / 代码 TODO
  • 即使只有模糊线索也要列出（对抗模型”偷懒”倾向）
  • 按 紧急 → 重要 → 普通 三级分类
  • 统一 Markdown 输出格式

后续在 Prompt 设计上更新了一些指令——“逐区域扫描”、”即使只有一条模糊的任务线索也要列出来”。这些指令是对模型”偷懒”倾向的对抗，实际测试中发现不加这些约束，模型经常直接回复”没有待办事项”。

日历同步

写入默认日历，无弹窗。

appt.Save() 直接入库，不像 .ics 方案需要弹出日历窗口让用户确认。事件会自动同步到手机——只要 iPhone/Android 上登录了同一个 Outlook/Exchange 账号。

add_to_calendar() 先通过 outlook.Name 探测 COM 是否真正可用，任何异常都静默降级到 .ics 方案。

COM 是微软的一套二进制接口标准，Windows 剪贴板、OLE 拖放、自动化等都建立在它之上。

设计原则

  • 零依赖云服务：不依赖任何外部账号体系。唯一的外部 API 是 DashScope（AI 分析）
  • 静默优先：后台运行时不弹控制台（clipboard_watcher.vbs 用 pythonw.exe 启动），成功/失败都走 Toast 通知，不打断用户工作流
  • 渐进降级：Outlook COM → .ics 文件 → 下载按钮。每条路径都有兜底，没有单点故障
  • 模块复用：calendar_sync.py 的 add_to_calendar() 同时服务于 hotkey_screenshot.py（后台监听）和 app.py（Web UI），一套逻辑两处复用
